# Orpheum 1.0.0 Release Notes

## Summary

Orpheum 1.0.0 is the first major release. The headline change is that the CLI now ships with the canonical catalog embedded, so `cargo install orpheum` works on a clean machine without a separate catalog checkout.

## Highlights

- Added the new `orpheum-catalog` crate to bundle the canonical scenarios, roles, workflows, artifacts, checks, and skill inventory into the installed CLI.
- Catalog-dependent commands now fall back to the embedded catalog after checking explicit `--catalog`, repo-local config, `ORPHEUM_CATALOG`, and runtime discovery.
- `orpheum init` and `orpheum update` now treat external catalog paths as optional overrides instead of required setup.
- Projects using the embedded catalog no longer need `.codex/orpheum/config.json`; that file is now only written when a project is intentionally pinned to an external catalog source.
- Added tests that run a copied binary outside the repo to validate fresh-machine behavior.

## Upgrade Notes

- Existing projects can continue to use repo-local config or `ORPHEUM_CATALOG` overrides if they intentionally follow an external development catalog.
- New installs can simply run `orpheum scenario list`, `orpheum init`, or `orpheum update` without first locating the Orpheum repo on disk.
- If an initialized project should stop following an external catalog checkout, rerun `orpheum update` without `--catalog` from a non-repo install and let it return to the embedded default.
- Session metadata still records the CLI version and will continue to warn when a newer CLI expects the project to refresh local guidance.

## Validation

- `cargo test -p orpheum-core`
- `cargo test -p orpheum --test cli`
