# Orpheum 0.3.0 Release Notes

## Summary

Orpheum 0.3.0 is a minor release focused on stronger artifact-scope discipline in slice planning and explicit refresh workflows for projects using newer CLI behavior.

## Highlights

- `delivery-slice-planning` now makes artifact layering first-class: enduring product and architecture artifacts stay broad, while bounded-slice detail is routed into backlog and planning artifacts.
- Product Owner, Solution Architect, and Technical Planner templates, workflows, and checks now explicitly guard against scope bleed between whole-project context and current-slice context.
- Active sessions now record the last Orpheum CLI version used in `.orpheum/session.json`.
- The CLI now warns when an active session was created by an older Orpheum version, or by a version that did not record CLI version metadata at all.
- `orpheum update` is now available as the explicit refresh command for updating local Orpheum guidance and session metadata in an initialized project.

## Upgrade Notes

- Existing projects should run `orpheum update` when warned that their active session was created by an older Orpheum CLI or does not record CLI version metadata.
- `orpheum init` still works as a refresh path, but `orpheum update` is now the clearer upgrade-oriented command for initialized projects.
- Consumers that inspect `.orpheum/session.json` should expect the optional `last_orpheum_cli_version` field.
- Teams using `delivery-slice-planning` should expect stronger guidance and checks around enduring product direction, enduring system architecture, and slice-local planning artifacts.

## Validation

- `cargo test -p orpheum --test cli`
- `cargo test -p orpheum-core`
