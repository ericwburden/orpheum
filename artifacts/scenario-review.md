# Scenario Review

## Purpose

Capture the explicit review posture for a drafted scenario package, including readiness, integration risks, conditions, and remediation decisions before the scenario is handed off for adoption or execution.

Use this artifact after the scenario definition and scenario integration map exist and before the scenario is treated as ready for downstream use.

This checked-in file is the canonical template definition. Create an instantiated project copy before filling it out for live Scenario Designer work.

## Completion Guidance

This artifact is complete when a downstream reader can understand what scenario package was reviewed, whether it is ready, conditional, or blocked, what integration risks still matter, and what remediation or follow-up is required.

## Review Scope

State which scenario, lifecycle window, and scenario package components were reviewed.

## Reviewed Inputs

Reference the scenario definition, scenario integration map, relevant role packages, and supporting notes or references used in this review.

## Overall Assessment

Summarize the current scenario posture and the strongest reasons for that assessment.

## Decision Status

State whether the scenario is ready, conditional, or blocked for downstream use and explain the basis for that judgment.

## Integration Risks And Failure Modes

Describe the most important coordination, sequencing, handoff, or boundary risks that still matter.

## Conditions And Remediation Decisions

Describe the conditions that must be satisfied, the remediation required, and where that remediation should be routed.

## Follow-Up Owners

Identify who owns each major follow-up, clarification, or package-strengthening action.

## Revisit Triggers

List the changes in scope, role support, evidence, tooling, or approval posture that should trigger another scenario review.

## Recommended Next Step

Describe the immediate next step, such as scenario handoff, role-package remediation, or scenario redesign.
