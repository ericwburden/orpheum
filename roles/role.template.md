# Role Template

## Purpose

Describe the repository or project context this role definition is designed for.

## Success Criteria

- State what a good outcome looks like.
- State what should be optimized for.
- State any constraints that matter more than speed.

## Working Principles

- Prefer small, reviewable changes.
- Preserve established conventions unless there is a clear reason to change them.
- Document assumptions that future agents would otherwise need to rediscover.

## Expected Inputs

- List the kinds of tasks this role is meant to support.
- Note any prerequisite documents, folders, or tools that should be checked first.

## Default Behavior

- Define how the agent should explore the codebase or project context.
- Define when to document before implementing.
- Define how to handle ambiguity and tradeoffs.

## Tooling Guidance

- List important tools, scripts, or skills to prefer.
- Note any tools that should be avoided or used only with justification.

## Change Standard

A good change under this guide should usually:

- solve a real problem
- leave durable documentation or structure behind
- reduce ambiguity for future contributors

## Validation

- Describe the checks the agent should run.
- Describe what to do if validation is unavailable or incomplete.

## Escalation Rules

- Explain when the agent should stop and ask for clarification.
- Explain what kinds of risky changes require explicit confirmation.

## Notes

Use this section for repo-specific conventions, vocabulary, or references.
